---
title: "Rue OpenTelemetry Traces"
---

Rue OpenTelemetry capture is **Runner-managed**, while persistence is **reporter-managed**. Traces exist only while a test function is executing inside an active OpenTelemetry-enabled test execution created by `Runner` or `uv run rue test --otel`.

## Enabling OpenTelemetry

### CLI

```bash
# Enable OpenTelemetry (the usual registered reporters also write local trace files)
uv run rue test --otel

# Disable OpenTelemetry explicitly, even if config enables it
uv run rue test --no-otel

# If you override reporters, keep OtelReporter attached for local trace files
uv run rue test --otel --reporter ConsoleReporter --reporter OtelReporter

# Disable content-bearing SUT/predicate attributes
uv run rue test --otel --no-otel-content
```

### Config

OpenTelemetry settings can be resolved from `rue.toml`, `[tool.rue]` in `pyproject.toml`, environment variables, CLI args, and `addopts`.

```toml
[tool.rue]
otel = true
otel-content = false
reporters = ["ConsoleReporter", "OtelReporter"]
```

Environment variable equivalents:
- `RUE_OTEL`
- `RUE_OTEL_CONTENT`

CLI flags override config and environment.

### Runner

```python
from rue.config import RueConfig
from rue.reports import ConsoleReporter, OtelReporter
from rue.testing import Runner

runner = Runner(
    config=RueConfig(otel=True, otel_content=True),
    reporters=[ConsoleReporter(), OtelReporter()],
)
```

If `config.reporters` is empty and you omit `reporters=...`, `Runner` uses all currently registered reporter instances. In a normal fresh process, that includes the built-in `ConsoleReporter` and `OtelReporter`.

## Public APIs

### `otel_span`

Create a custom child span inside the current OpenTelemetry-enabled test.

```python
from rue import otel_span

def test_pipeline():
    with otel_span("retrieve", {"query": "weather"}):
        pass
```

Outside an active OpenTelemetry-enabled test execution, `otel_span()` is a no-op and does not create Rue span output.

### `OtelTrace`

Declare `otel_trace` in a test signature to inspect the current OpenTelemetry test session.

```python
import rue

def test_agent_workflow(my_agent, otel_trace: rue.OtelTrace):
    result = my_agent("hello")

    with rue.otel_span("postprocess"):
        assert result

    spans = otel_trace.get_child_spans()
    llm_calls = otel_trace.get_llm_calls()
    otel_trace.set_attribute("response.length", len(result))
```

Available properties and methods:
- `otel_trace_id`
- `otel_span_id`
- `get_child_spans()`
- `get_llm_calls()`
- `get_sut_spans(name=None)`
- `set_attribute(key, value)`

`otel_trace` is only available while an OpenTelemetry-enabled test is actively executing. If OpenTelemetry is off, resolving it raises at runtime.

## Behavior

- Rue creates one root span per OpenTelemetry-enabled test execution: `test.<full_name>`.
- Aggregator executions (`repeat`, `parametrize`, `iter_cases`, `iter_case_groups`) do not get their own OpenTelemetry roots; each child execution gets its own trace.
- `@rue.sut` and `@predicate` write spans only when an active OpenTelemetry session exists.
- OpenAI and Anthropic client instrumentation is initialized by the OpenTelemetry runtime and captured only when calls belong to an active OpenTelemetry test session.

## Storage

- Enable capture with `RueConfig(otel=True)`, `--otel`, or config files; persistence still happens only when an `OtelReporter` is attached
- In the usual fresh-process setup, `uv run rue test --otel` writes `.rue/traces/<run_id>/<execution_id>.json` because the built-in `OtelReporter` is registered by default
- `OtelReporter` stores compact JSON payloads under `.rue/traces/<run_id>/<execution_id>.json`
- `OtelReporter` keeps only the 5 most recent run directories under `.rue/traces`
- Custom OTEL persistence or forwarding belongs in custom reporters via `Reporter.on_trace_collected()`
- File format: one compact JSON trace payload per execution from `session.serialize()` with `run_id`, `execution_id`, and `spans`
- SQLite does not persist OpenTelemetry trace payloads or trace IDs

`otel_content` controls Rue-added content-bearing SUT and predicate attributes:
- `sut.input.args`
- `sut.input.kwargs`
- `sut.output`
- `predicate.input.actual`
- `predicate.input.reference`
- `predicate.message`

When `otel_content=false`, Rue records only coarse metadata for those spans.
