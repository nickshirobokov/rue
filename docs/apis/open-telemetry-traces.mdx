---
title: "Rue OpenTelemetry Traces"
---

Rue OpenTelemetry capture is **Runner-managed**, while persistence is **reporter-managed**. Traces exist only while a test function is executing inside an active OpenTelemetry-enabled test execution created by `Runner` or `uv run rue test`.

## OpenTelemetry Defaults

### CLI

```bash
# OpenTelemetry is enabled by default
uv run rue test

# Disable OpenTelemetry explicitly, even if config enables it
uv run rue test --no-otel

# Force-enable OpenTelemetry if config or environment disabled it
uv run rue test --otel

# If you override reporters, keep OtelReporter attached for local trace files
uv run rue test --reporter ConsoleReporter --reporter OtelReporter
```

### Config

OpenTelemetry settings can be resolved from `rue.toml`, `[tool.rue]` in `pyproject.toml`, environment variables, CLI args, and `addopts`.

```toml
[tool.rue]
otel = false
reporters = ["ConsoleReporter", "OtelReporter"]
```

Environment variable:
- `RUE_OTEL`

CLI flags override config and environment.

### Runner

```python
from rue.config import RueConfig
from rue.reports import ConsoleReporter, OtelReporter
from rue.testing import Runner

runner = Runner(
    config=RueConfig(),
    reporters=[ConsoleReporter(), OtelReporter()],
)
```

Use `RueConfig(otel=False)` to disable capture programmatically.

If you do not set an explicit reporter list, a normal fresh process uses the built-in `ConsoleReporter` and `OtelReporter`.

If you set `config.reporters` or pass `--reporter` and leave out `OtelReporter`, OpenTelemetry capture still runs, but local `.rue/traces/...` files are not written.

## Public API

### `SUT` trace access

OpenTelemetry data is exposed only through `SUT` instances while a test is running.

```python
import rue

@rue.resource.sut
def my_agent():
    return rue.SUT(call_agent)

def test_agent_workflow(my_agent):
    result = my_agent.instance("hello")

    assert result
    assert my_agent.get_sut_spans()
    assert my_agent.get_child_spans()
    assert my_agent.get_llm_calls()
```

Available properties and methods:
- `get_sut_spans()`
- `get_child_spans()`
- `get_llm_calls()`

`get_sut_spans()`, `get_child_spans()`, and `get_llm_calls()` return `[]` until that `SUT` has been called in the current execution.

## Behavior

- Rue creates one root span per OpenTelemetry-enabled test execution: `test.<full_name>`.
- Aggregator executions (`iterate`, `iterate.params`, `iterate.cases`, `iterate.groups`) do not get their own OpenTelemetry roots; each child execution gets its own trace.
- `@rue.resource.sut` and `@predicate` write spans only when an active OpenTelemetry session exists.
- Full traces are not exposed to test code during execution; test code can inspect only the subtree of each `SUT`.
- OpenAI and Anthropic client instrumentation is initialized by the OpenTelemetry runtime and captured only when calls belong to an active OpenTelemetry test session.

## Storage

- Capture is enabled by default; persistence still happens only when an `OtelReporter` is attached
- In the usual fresh-process setup, `uv run rue test` writes `.rue/traces/<run_id>/<execution_id>.json` because the built-in `OtelReporter` is part of the default reporter set
- If you override the reporter list and omit `OtelReporter`, Rue captures traces but does not persist local trace files
- `OtelReporter` stores compact JSON payloads under `.rue/traces/<run_id>/<execution_id>.json`
- `OtelReporter` keeps only the 5 most recent run directories under `.rue/traces`
- Custom OTEL persistence or forwarding belongs in custom reporters via `Reporter.on_trace_collected(tracer, execution_id)`
- File format: one compact JSON trace payload per execution with `run_id`, `execution_id`, and a nested `trace` tree rooted at the test span
- `session.serialize()` stays flat with `run_id`, `execution_id`, and `spans` for internal use and custom reporters
- SQLite does not persist OpenTelemetry trace payloads or trace IDs

When OpenTelemetry is active, Rue records these content-bearing SUT and predicate attributes:
- `sut.input.args`
- `sut.input.kwargs`
- `sut.output`
- `predicate.input.actual`
- `predicate.input.reference`
- `predicate.message`
