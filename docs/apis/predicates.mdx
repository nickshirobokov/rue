## `predicate`

Decorate a sync or async comparison function so Rue records predicate metadata for assertions and traces.

```python
from rue import predicate
from rue.predicates import PredicateResult


@predicate
def my_predicate(actual: str, reference: str, ...) -> bool | PredicateResult:
    ...


@predicate(name="custom_name")
def my_predicate(actual: str, reference: str, ...) -> bool | PredicateResult:
    ...
```

### Parameters

| Name | Type | Description |
| --- | --- | --- |
| `func` | `Callable` | Sync or async function returning `bool` or `PredicateResult` |
| `name` | `str \| None` | Optional display name for reports and traces |

### Rules

- The function must declare named `actual` and `reference` parameters.
- Those parameters may be positional-or-keyword or keyword-only.
- Positional-only `actual` and `reference` are rejected.
- The wrapped call returns `bool` to the caller.

### Recording behavior

If the wrapped function returns `bool`, Rue creates a `PredicateResult` from the bound call.

- `strict` defaults to `False` if your function does not declare it.
- `confidence` defaults to `1.0` if your function does not declare it.
- `message` defaults to `None` if your function does not declare it.

If the wrapped function returns `PredicateResult`, Rue uses that object as-is.

## `PredicateResult`

Structured metadata for a predicate evaluation.

```python
from rue.predicates import PredicateResult
```

### Fields

| Name | Type | Description |
| --- | --- | --- |
| `actual` | `str` | Recorded observed value |
| `reference` | `str` | Recorded comparison value |
| `name` | `str` | Predicate name |
| `strict` | `bool` | Recorded strictness flag |
| `confidence` | `float` | Confidence in `[0, 1]` |
| `value` | `bool` | Boolean verdict |
| `message` | `str \| None` | Optional explanation |

`PredicateResult` is a Pydantic model. It is commonly produced internally by Rue, and you can also return it directly from a decorated predicate.

## Built-in predicates

All built-in predicates are async callables imported from `rue.predicates`. They always return `bool`.

### Shared parameters

| Name | Type | Description |
| --- | --- | --- |
| `actual` | `str` | Document being evaluated |
| `reference` | `str` | Comparison document, topic string, policy string, or sample depending on the predicate |
| `strict` | `bool` | Changes semantics for the factual family; accepted but semantically ignored for the non-factual family |
| `with_explanation` | `bool` | When `True`, ask the built-in predicate to record an explanation in `PredicateResult.message`; the awaited return value is still `bool` |

### Factual family

#### `has_facts`

Returns whether every fact in `reference` is supported by `actual`.

```python
async def has_facts(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is a reference document whose facts must be covered.
- `strict=False` uses open-world evaluation.
- `strict=True` uses closed-world evaluation.

#### `has_unsupported_facts`

Returns whether `actual` contains at least one fact not supported by `reference`.

```python
async def has_unsupported_facts(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is the grounding document.
- `True` means `actual` added unbacked factual content.

#### `has_conflicting_facts`

Returns whether any fact in `actual` conflicts with any fact in `reference`.

```python
async def has_conflicting_facts(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is the document to compare against for contradiction.
- A single conflict is enough for `True`.

#### `matches_facts`

Returns whether `actual` and `reference` are factually equivalent.

```python
async def matches_facts(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is the comparison document.
- This is the strongest factual predicate: two-way support and no conflict.

### Non-factual family

#### `matches_writing_style`

Returns whether `actual` and `reference` are written in materially the same style.

```python
async def matches_writing_style(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is a writing sample whose style is the target style.
- `strict` is accepted today but does not change the predicate's meaning.

#### `matches_writing_layout`

Returns whether `actual` and `reference` use materially the same structural layout or template.

```python
async def matches_writing_layout(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is a reference document, schema instance, or template sample.
- `strict` is accepted today but does not change the predicate's meaning.

#### `has_topic`

Returns whether `actual` is substantively about the topic described by `reference`.

```python
async def has_topic(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is a topic specification string such as `"refund policy"` or `"database performance"`.
- `strict` is accepted today but does not change the predicate's meaning.

#### `follows_policy`

Returns whether `actual` satisfies the rule or constraints described by `reference`.

```python
async def follows_policy(
    actual: str,
    reference: str,
    strict: bool = False,
    with_explanation: bool = False,
) -> bool
```

- `reference` is a policy specification string such as `"must be valid JSON"` or `"must not mention pricing"`.
- `strict` is accepted today but does not change the predicate's meaning.

## Example

```python
from rue import predicate
from rue.predicates import (
    follows_policy,
    has_conflicting_facts,
    has_unsupported_facts,
)


@predicate
def contains_order_id(actual: str, reference: str) -> bool:
    return reference in actual


async def test_order_reply():
    source = "Order ORD-2041 has shipped and should arrive by Friday."
    reply = "Hi Sam,\n\nOrder ORD-2041 has shipped and should arrive by Friday.\n\nBest,\nSupport"

    assert contains_order_id(reply, "ORD-2041")
    assert not await has_unsupported_facts(reply, source, with_explanation=True)
    assert not await has_conflicting_facts(reply, source)
    assert await follows_policy(reply, "must include a signoff")
```
