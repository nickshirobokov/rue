Predicates are boolean comparison functions in Rue. They answer one narrow question about an `actual` value relative to a `reference` value, and they are designed to read naturally inside `assert` statements.

<Note>
Behavioral semantics for built-in predicates are defined by
[src/rue/predicates/SPEC.md](https://github.com/NickSherrow/rue/blob/main/src/rue/predicates/SPEC.md).
Use this page as the overview, and treat the spec as the source of truth.
</Note>

## What a predicate is

A predicate in Rue:

- returns `bool`
- compares `actual` to `reference`
- answers one specific question instead of producing a broad score
- can run inside normal tests, metrics, traces, and assertion rewriting

That narrowness matters. `matches_facts`, `matches_writing_style`, `has_topic`, and `follows_policy` are intentionally different checks, even if they operate on the same pair of documents.

## Two ways to work with predicates

### Built-in predicates

Rue ships 8 built-in predicates in `rue.predicates`. They are async, LLM-backed, and split into two families.

| Family | Predicate | Question it answers |
| --- | --- | --- |
| Factual | `has_facts` | Does `actual` cover every fact in `reference`? |
| Factual | `has_unsupported_facts` | Does `actual` add any fact that `reference` does not support? |
| Factual | `has_conflicting_facts` | Does `actual` contradict `reference` anywhere? |
| Factual | `matches_facts` | Are `actual` and `reference` factually equivalent? |
| Non-factual | `matches_writing_style` | Are they written in materially the same style? |
| Non-factual | `matches_writing_layout` | Do they use materially the same structure or template? |
| Non-factual | `has_topic` | Is `actual` substantively about the topic in `reference`? |
| Non-factual | `follows_policy` | Does `actual` satisfy the rule in `reference`? |

Use built-ins when your comparison already matches one of those questions.

### Decorated predicates

Use `@predicate` when you need a custom comparison that Rue does not ship out of the box.

```python
from rue import predicate


@predicate
def contains_ticket_id(actual: str, reference: str) -> bool:
    return reference in actual
```

The decorator makes your function participate in Rue's predicate recording and tracing machinery while keeping the call site simple:

```python
def test_ticket_response():
    response = "Ticket INC-1042 is now closed."
    assert contains_ticket_id(response, "INC-1042")
```

## What Rue records

When a predicate runs inside a Rue-transformed `assert`, Rue records a `PredicateResult` for reporting, storage, and tracing.

- Built-in predicates still return `bool` to your test.
- Decorated predicates still return `bool` to your test.
- Trace spans are named `predicate.<predicate_name>`.
- Recorded metadata includes the predicate name, `actual`, `reference`, `strict`, `confidence`, the boolean value, and an optional message.

This is why predicate docs need both an API view and a semantic view: the Python call returns a simple boolean, but the evaluation still produces structured metadata behind the scenes.
