<Note>SUT stands for *System Under Test*.</Note>
<Note>`@rue.resource.sut` is optional. It exists to make your agent/workflow callable injectable and OpenTelemetry-observable.</Note>

`@rue.resource.sut` registers a **SUT factory function** as an injectable **resource**. The factory must return (or yield) a `rue.SUT(...)` object, and each invocation of `sut.instance` (or a wrapped method on it) is wrapped in an OpenTelemetry span. This gives you three things:
- A clean dependency injection (DI) boundary in your tests (you call the injected parameter's `instance`, not a global).
- SUT-owned stdout/stderr capture you can assert on through `sut.stdout`, `sut.stderr`, and `sut.captured_output`.
- OpenTelemetry spans you can query from that `SUT` inside a test (when OpenTelemetry is enabled).

```python
import rue
from rue import SUT

from demo_app.chatbot import chatbot as prod_chatbot

@rue.resource.sut
def chatbot():
    return SUT(prod_chatbot)

def test_chatbot_works(chatbot):
    out = chatbot.instance("hello")
    assert "hello" in out
    assert "hello" in chatbot.stdout.text
```

## Basic Usage
The most common use case for `@rue.resource.sut` is asserting on the SUT call span(s).

<Note>
SUT trace accessors work only when OpenTelemetry is enabled. Rue enables it by default in `uv run rue test`; use `--no-otel` to opt out.
</Note>

<Steps>
<Step title="Decorate the system under test">

```python
# tests/test_agent.py
import rue
from rue import SUT

from demo_app.weather import weather_agent as prod_weather_agent

@rue.resource.sut
def weather_agent():
    return SUT(prod_weather_agent)
```
</Step>
<Step title="Inject into a `test` and assert on OpenTelemetry">

```python
# tests/test_agent.py
import rue

@rue.test
def test_weather_agent_calls_tools(weather_agent):
    out = weather_agent.instance("What's the weather in SF?")

    spans = weather_agent.root_spans
    assert spans
    assert spans[0].attributes.get("rue.sut.name") == "weather_agent"
    assert weather_agent.all_spans
```
</Step>
</Steps>

## What `@rue.resource.sut` actually does

### Injection semantics
`@rue.resource.sut` registers a resource factory so the Rue runner can inject it into tests by parameter name (by default, a **case-scoped** resource). The injected value is a `SUT` object.

If constructing your SUT is expensive, you can widen its lifecycle using `scope` (the same values as `@rue.resource`):

```python
import rue
from rue import SUT
from demo_app.weather import weather_agent as prod_weather_agent

@rue.resource.sut(scope="process")
def weather_agent():
    return SUT(prod_weather_agent)
```

That’s why you should call the injected `chatbot.instance` inside `test_chatbot_works(chatbot)`, not the decorated global name.

### Naming rules (important for DI and tracing)
- **SUT name**: the factory function name (`def weather_agent(): ...` → `"weather_agent"`).
  - Pick your factory name intentionally, since it’s used for dependency injection *and* the `sut.<sut_name>.<method_name>` span name.

### Instance-based SUTs (span a method)
Use `SUT(instance, methods=["run"])` when the thing you want to exercise is an instance method instead of `__call__`:

```python
import rue
from rue import SUT

from demo_app.agents import WeatherAgent

@rue.resource.sut
def weather_agent():
    return SUT(WeatherAgent(), methods=["run"])

def test_custom_method(weather_agent):
    out = weather_agent.instance.run("task")
    assert out
```

## SUT-owned output capture
Each wrapped SUT call appends Python-level stdout/stderr writes to the `SUT` instance:

- `sut.stdout.text`
- `sut.stderr.text`
- `sut.captured_output.combined.text`
- `sut.captured_output.events`

Use `sut.clear_output()` when you want to reset accumulated output manually.

```python
import sys
import rue
from rue import SUT


class Agent:
    def run(self, prompt: str) -> str:
        print(f"running: {prompt}")
        sys.stderr.write("debug: tool invoked\n")
        return prompt.upper()


@rue.resource.sut
def agent():
    return SUT(Agent(), methods=["run"])


def test_agent_output(agent):
    assert agent.stdout.text == ""

    assert agent.instance.run("hello") == "HELLO"
    assert "running: hello" in agent.stdout.text
    assert "debug: tool invoked" in agent.stderr.text
```

## SUT span attributes
Each SUT call creates a span named `sut.<sut_name>.<method_name>` and sets:
- `rue.sut=true`
- `rue.sut.name=<sut_name>`
- `rue.sut.method=<method_name>`

Rue also records:
- `sut.input.args` / `sut.input.kwargs` (`repr(...)`)
- `sut.output` (`repr(...)`)

## Recommendations

### 1. Create isolated helpers; don't touch your production code

Many evaluation frameworks require developers to modify their production codebase to instrument OpenTelemetry traces. Rue avoids this pattern. The best way to introduce SUTs to your suite is to create isolated wrapper functions within your `test_*.py` files.

**Don't do this:**
```python
# src/app/agent.py
import rue
from typing import Literal

@rue.resource.sut
def agent():
    # Don't decorate production entrypoints; wrap them in your tests layer instead.
    ...
```

**Do this:**
```python
# tests/test_agent.py
import rue
from rue import SUT

from functools import partial
from app import agent as prod_agent

@rue.resource.sut
def marketing_agent():
    """Q&A system that answers questions about marketing concepts."""
    return SUT(partial(prod_agent, domain="marketing"))

@rue.test
def test_marketing_agent_invokes(marketing_agent):
    out = marketing_agent.instance("What's CAC?")
    assert out
```

### 2. Pass using Dependency Injection; don't call directly

`@rue.resource.sut` registers a resource factory for injection. Calling the decorated global directly is the wrong pattern (and may not work the way you expect). Always call the injected parameter.

**Don't do this:**
```python
import rue
from rue import SUT
from demo_app.chatbot import chatbot as prod_chatbot

@rue.resource.sut
def chatbot():
    return SUT(prod_chatbot)

def test_chatbot_runs():
    # This is the resource factory, not the injected callable
    sut = chatbot()
    out = sut.instance("Hello!")
    assert out
```

**Do this:**
```python
import rue
from rue import SUT
from demo_app.chatbot import chatbot as prod_chatbot

@rue.resource.sut
def chatbot():
    return SUT(prod_chatbot)

def test_chatbot_runs(chatbot):
    out = chatbot.instance("Hello!")
    assert out
```
